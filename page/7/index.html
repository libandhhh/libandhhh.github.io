蓝桥杯--最大比例

Share

内容简介 : 蓝桥杯历年题目

  • 试题内容 — 最大比例

    X星球的某个大奖赛设了 MM 级奖励。

    每个级别的奖金是一个正整数。

    并且，相邻的两个级别间的比例是个固定值。

    也就是说：所有级别的奖金数构成了一个等比数列。

    比如：16,24,36,5416,24,36,54，其等比值为：3/23/2。

    现在，我们随机调查了一些获奖者的奖金数。

    请你据此推算可能的最大的等比值。

    输入格式

    第一行为数字 NN ，表示接下的一行包含 NN 个正整数。

    第二行 NN 个正整数 XiXi，用空格分开，每个整数表示调查到的某人的奖金数额。

    输出格式

    一个形如 A/BA/B 的分数，要求 A、BA、B 互质，表示可能的最大比例系数。

    数据范围

    0<N<1000<N<100
    0<Xi<10120<Xi<1012
    数据保证一定有解。

    输入样例1：

    3
1250 200 32

    输出样例1：

    25/4

    输入样例2：

    4
3125 32 32 200

    输出样例2：

    5/2

    输入样例3：

    3
549755813888 524288 2

    输出样例3：

    4/1
  • 我的代码
#include<iostream>
#include<cstring>
#include<cstdio>
#include<algorithm>
using namespace std;
const int N = 110;
typedef long long LL;
LL x[N],a[N],b[N];
int n;

//辗转相减法
LL gcd(LL a,LL b)
{
    return b ? gcd(b,a%b) : a; 
}
//更相减损术
LL gcd_sub(LL a,LL b)
{
    if(a < b) swap(a,b);
    if(b == 1) return a;
    return gcd_sub(b,a/b);
}

int main()
{
    cin >> n;
    for(int i = 0; i < n; i++)
    {
        cin >> x[i];
    }
    sort(x, x + n);
    int cnt = 0;
    for(int i = 1; i < n; i++)
    {
        if(x[i] != x[i - 1])
        {
            LL d = gcd(x[i],x[0]);
            a[cnt] = x[i] / d;
            b[cnt] = x[0] / d;
            cnt ++;
        }
    }

    LL up = a[0],down = b[0];

    for(int i = 1; i < cnt; i++)
    {
        up = gcd_sub(up,a[i]);
        down = gcd_sub(down,b[i]);
    }
    cout << up << "/" << down << endl;
    return 0;
}

Mac之中如何解决安装问题

Share

内容简介 : 对于很多mac新手以及很多使用win的大部分用户来说,mac相比于很多操作系统,除了超级舒适的操作系统,以及便捷的代码命令工具,也有很多让人头疼的地方,由于mac对于用户的安全保护性,往往可能不会识别一些不经过苹果开发团队认可的软件,但是我们用户又可能十分需要这些软件,因此就会出现无法安装的问题,因此,小编就安利一波解决方法

1.找到系统偏好设置

打开安全性与隐私 (ps:快捷键是’commad+空格’直接搜索系统偏好设置即可)

2.打开我的iterm(或者可以打开mac的终端)
输入命令行:sudo spctl –master-disable (sudo代表的是进入管理员模式)
这里输入的命令可以打开mac隐藏的任何来源的选项,这样就可以随心所欲的进行安装了,但是注意安全性哦！！！

3.点开小锁图标,输入你的mac用户名和密码即可

4.点击任何来源就可以愉快的下载了。

Git常见的指令操作

Share

内容简介 : 学习git的常规操作

1.对于一个初始化的项目来说我们使用git来管理进行版本操作是十分简易便捷的,对于大多数的大型管理应用来说,git这种版本工具十分适合大型企业的项目维护,因此gitgub作为最大的代码平台也因此吸引了许许多多的企业甚至大牛,营造了如今十分火热的生态圈,学会使用git的基本操作对开发人员来说,是必不可少的一项技能,言归正传,我们拿到一个项目如何进行管理。

2.项目起手

2.1在本地进行创建本地仓库 

$ git init --创建一个当地的维护仓库

2.2将要上传的项目进行添加草版本库之中

$ git add . -- 默认将所有的文件添加到缓存区之中
$ git add <file name>  -- 将指定默认的文件添加到缓存区中

2.3进行第一次提交 书写合适的提交信息

$ git commit -m "我的第一次提交"

2.4进行远程连接

$ git remote add origin SSH -- 这里的SSH就是连接仓库的秘钥,在github的仓库里就能找到

2.5.进行推送

$ git push -u origin master -- 推送到仓库默认的master分支里面去

3.项目维护与更新

直接上操作

$ cd text --进入上传的文件夹
$ git add . --添加文件
$ git commit -m "第二次提交"
$ git pull origin master  --拉取分支
$ git push -u origin master --上传文件

还有很多的git指令可以使用 git -help来进行查询查看即可

4.自己使用的错误要点:

自己搭建的hexo博客是自动生成静态文件,并且也是挂载到github上面的,但是由于对git的工作运行理念了解的不清楚,导致在第一次提交后打算进行更新远程仓库时候,居然无脑的把自己的public文件删掉了,重新来了一遍hexo g生成,这样导致本地仓库与远程仓库的时间不一致的问题使得无法进行pull操作,导致merge也错误了。因此多使用git操作一定是会提升以后的工作开发效率的孰能生巧鸭。

蓝桥杯--带分数

Share

内容简介 : 蓝桥杯历年题目

  • 试题内容 — 带分数

    100100 可以表示为带分数的形式：100=3+69258714100=3+69258714

    还可以表示为：100=82+3546197100=82+3546197

    注意特征：带分数中，数字 1∼91∼9 分别出现且只出现一次（不包含 00）。

    类似这样的带分数，100100 有 1111 种表示法。

    输入格式

    一个正整数。

    输出格式

    输出输入数字用数码 1∼91∼9 不重复不遗漏地组成带分数表示的全部种数。

    数据范围

    1≤N<106

    输入样例1：

    100

    输出样例1：

    11

    输入样例2：

    105

    输出样例2：

    6
  • 我的代码
#include<iostream>
#include<cstdio>
#include<cstring>
#include<algorithm>
using namespace std;
const int N = 10; //枚举10位数
typedef long long LL;

int n;
bool st[N],backup[N];
int ans;

bool check(int a,int c)
{
    //直接算出b的值就好了   考虑到c是八位数 因此和c相乘的话可能会爆掉int
    LL b = n * (LL) c - a * c;

    //特殊判断一下  看一下是否有没有找到的  保证每一位都要有的
    if(!a || !b || !c) return false;
    memcpy(backup,st, sizeof st); //搞一个特判数组 看是否满足
    //拿出 b 的每一位看一看是否满足没有查询到的操作
    while(b)
    {
       int x =  b % 10; //一直拿掉b的最后一位
       b /= 10;
       if( !x || backup[x] ) return false;
       backup[x] = true;
    }
    //判断是否每一个都有用到
    for(int i = 1;i <= 9; i++)
        if(!backup[i])
            return false;

    return true;
}

void dfs_c(int u,int a,int c)
{
    //通过 a 和 c 来算出 b
    if(check(a,c)) ans ++; //通过 a c 来判断是否符合

    for(int i = 1; i <= 9; i++)
    {
        if(!st[i])
        {
            st[i] = true;
            dfs_c(u+1,a,c * 10 + i);
            st[i] = false; //恢复现场
        }
    }
}

void dfs_a(int u,int a)
{
    if(a >= n) return;
    if(a)  dfs_c(u,a,0);  //对于枚举好的a来说 依次进行枚举 c

    //对于a进行每一位的枚举
    for(int i = 1; i <= 9; i++)
    {
        if(!st[i])
        {
            st[i] = true;
            dfs_a(u + 1,a * 10 + i);
            st[i] = false;
        }
    }

}
int main()
{
    cin >> n;
    dfs_a(0,0);
    cout << ans << endl;
    return 0;
}

搭建博客遇到的坑点

Share

内容简介 : 记录自己搭建博客遇到的坑点以及建议

搭建的第一次博客,感到十分无力鸭,踩过的坑是在太多了.

1.第一次下载hexo的主题文件时候,虽然在本地端口4000成功跑起来了,但是还是十分无脑。使用命令行十分生疏。建议后来者多看看hexo官方的doc，以便可以更好的避过坑点。(附上官方链接: https://hexo.io/)

2.先跑起来再说,对于theme的选择要合适,建议多选择生态圈比较繁荣的theme主体,这样在遇到问题的时候可以更加方便查找,省去很多的力气等等。

3.没有看懂hexo的生成文件是另一大痛点,由于模块化的组装方式,hexo里面很多的文件都是.ejs的js模板文件,对会一些前端的朋友还是比较友好的。最好搞清楚hexo的文件架构以便可以进行个性化的定制操作。建议了解一下ejs的语法,这样就能更加得心应手的看懂代码了。(附上ejs官方链接: https://ejs.bootcss.com/)

4.自己还很无脑的不知道如何进行插件查询和使用,小编自己使用过的一个hexo-tagcloud标签云插件就很无脑,记得要在自己写的md文件中加上categories分类和tags这样就可以使用了,不要无脑进行查找。(附上hexo插件的链接: https://hexo.io/plugins/)

5.还有使用一个统建站点访问记录的一个插件,详情请见不蒜子的博客,很方便的哦,但是不要在本地跑,本地跑会失灵的。

6.说在后面: 万事开头难,坚持才能行得通哦。
